## 一、交付流程

### 步骤

1. 写前分析（阶段1）
2. 写作（阶段2）
3. 自检（阶段3）+ 文笔润色（阶段3.5）
4. 修订（阶段4）
5. 章末元数据输出（阶段5）
6. 安全过滤：`guard.py filter chapters/ "第*章*.md"`
7. 合并MD → 转docx（pypandoc / pandoc CLI）
8. 发前审核：`guard.py scan merged.docx`
9. 发送邮件
10. 清理临时文件：`guard.py clean chapters/`

### 文件命名

- 正文：`第X章_章节名.md`
- 锚点：`第X章_锚点.md`
- 草稿：`_tmp_第X章_*.md`（写完后删除）
- 场景卡：`第X章_场景卡.md`

### docx转换

优先 pypandoc（`pip install pypandoc_binary`），备选 pandoc CLI：

```python
import pypandoc
pypandoc.convert_text(md_text, 'docx', format='md', outputfile='output.docx')
```

## 二、状态追踪文件结构

建议在项目根目录维护以下文件：

```
project/
├── outline.md              # 大纲
├── state_tracking.md       # 状态追踪
├── timeline.md             # 时间线
├── characters.md           # 人物库
└── foreshadowing.md        # 伏笔表
```

### state_tracking.md 示例

```markdown
# 状态追踪

## 角色状态
| 角色 | 位置 | 情绪 | 物品 | 身体 |
|------|------|------|------|------|
| 沈默 | 拾遗酒馆 | 沉淀 | 账本 | 无伤 |

## 时间线
- 第64天：沈默在账本封面写"拾遗录"
- 第65天：深色衣服的人在补给站打听

## 伏笔
| ID | 内容 | 状态 | 回收章 |
|----|------|------|--------|
| F001 | 前文明浇铸墙 | active | - |
```

## 三、场景构建模板

每个场景建议覆盖：

```
- 环境锚点（空间位置、时间、光源、声音、气味）
- 人物动作/对话（推进信息或情绪）
- 情绪节拍（一个场景内从A→B的情绪变化）
- 信息推进（至少一条新信息或一个旧信息的新角度）
```

## 四、常见问题

### 主角知道太多怎么办？
不要让他"一眼看穿"。他的知识和经验是不错的起点，但泰拉世界的规则是另一套逻辑。让他的判断部分正确、部分错误。

### 配角没存在感怎么办？
每个配角出场时问自己：他现在有自己的动机吗？他最近在忙什么？他走进拾遗不是为了喝茶，是为了什么？

### 写不动怎么办？
不要硬写。写一段过渡场景：描写天气、描写路人、描写一条狗。有时候最好的入口不是情节入口，是感官入口。
